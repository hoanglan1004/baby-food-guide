## 1. Overview

### 1.1 Design Goals

1. **"고민 제로"** — 앱 열면 오늘 뭘 먹일지 바로 보임. 검색/탐색 불필요
2. **시각 우선** — 텍스트 최소화, 모든 정보를 색상/아이콘/일러스트로 전달
3. **원핸드 조작** — 아기 안고 한 손으로 조작 가능
4. **오프라인 완전 지원** — 마트, 외출 중 인터넷 없이 100% 동작
5. **과학적 신뢰** — 모든 정보에 출처 표시 (WHO, AAP, 소아과학회)

### 1.2 Design Principles

- **Less is More**: 화면당 정보 1가지. 스크롤보다 탭/스와이프
- **Visual First**: 텍스트 설명 전에 항상 시각 요소가 먼저
- **Trust by Source**: 정보마다 출처 뱃지 표시로 신뢰감
- **Progressive Disclosure**: 기본은 간단히, 탭하면 상세 펼침

## 4. 주간 식단 추천 엔진 (핵심 기능)

### 4.1 설계 원칙

```
엄마의 고민: "오늘 뭘 먹이지?"
     ↓
앱의 답: "이번 주는 이거예요. 오늘은 이거 먹이세요."
     ↓
따라만 하면 → 영양 균형 + 새 식재료 안전 도입 + 알레르기 관찰 완료
```

### 4.2 식단 생성 로직

```
입력: 하윤이 생일(2025-10-06) + 오늘 날짜
  ↓
1단계: 월령 계산 → 현재 단계 결정 (초기/중기/후기/완료기)
  ↓
2단계: 이유식 시작일(생후 6개월) 기준 → 몇 주차인지 계산
  ↓
3단계: 해당 주차의 식단 템플릿 로드 (meal-plans.json)
  ↓
4단계: 이미 시도한 식재료(일지 기록) 참고 → 식단 조정
  ↓
출력: 이번 주 7일 식단 + 오늘의 이유식 + 장보기 목록
```

### 4.3 새 식재료 도입 규칙 (3일 규칙)

```
[새 식재료 도입 흐름]

Day 1: 🆕 새 식재료 소량 시도 (1숟가락)
       → 반응 기록 필수
Day 2: 같은 식재료 시도 (양 늘리기)
       → 반응 기록
Day 3: 같은 식재료 시도 (정상량)
       → 반응 기록
Day 4: ✅ 이상 없으면 → "안전 식재료"로 등록
       🚨 이상 있으면 → "주의 식재료"로 표시, 2주 후 재시도 권장
```

### 4.4 단계별 식단 구성

| 단계 | 월령 | 하루 횟수 | 구성 | 새 식재료 |
|------|------|----------|------|----------|
| **초기** | 6개월 | 1회 | 미음(곡류) + 채소/과일 퓨레 1종 | 주 1-2종 |
| **중기** | 7-8개월 | 2회 | 죽 + 채소 + 단백질 | 주 1-2종 |
| **후기** | 9-11개월 | 3회 | 진밥 + 채소 + 단백질 + 과일 | 주 1종 |
| **완료기** | 12개월+ | 3회+간식 | 일반식 전환 | 자유롭게 |

### 7.2 Service Worker 캐싱 전략

```
Cache First (오프라인 우선):
  ├── HTML 페이지 (6개)
  ├── CSS 파일
  ├── JS 파일
  ├── JSON 데이터 (식재료, 레시피, 식단)
  ├── 아이콘/이미지
  └── manifest.json

Network First (온라인 시 최신 버전):
  └── sw.js (Service Worker 자체)

localStorage (캐시 아님, 사용자 데이터):
  ├── 이유식 일지
  ├── 시도한 식재료 기록
  └── 아기 생일 설정
```
